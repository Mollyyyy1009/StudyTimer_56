# 项目设计说明
## 项目背景与目标

**一、项目背景**

​	在信息爆炸与碎片化时代，学习场景中常面临注意力分散、时间规划模糊、学习效率低下等痛点——社交媒体弹窗、无关信息干扰频繁打断专注状态，缺乏科学的时间管理工具导致学习节奏混乱，目标拆分不清晰则容易引发畏难情绪。同时，传统计时工具仅具备基础计时功能，无法匹配学习场景下“专注-休息”的循环需求，也难以实现学习数据追踪与进度可视化，难以帮助用户建立稳定的学习习惯。为解决上述问题，契合当下高效学习、量化成长的需求，StudyTimer项目应运而生。

**二、项目目标**

1. 核心功能目标：开发一款集专注计时、番茄工作法适配、自定义时段设置于一体的工具，支持“专注模式”与“休息模式”智能切换，屏蔽无关干扰，助力用户快速进入深度专注状态。
2. 体验优化目标：打造简洁易用、低学习成本的界面，适配多终端使用场景，支持数据云同步，让用户在手机、平板、电脑端均可无缝切换使用。
3. 数据价值目标：实现学习数据（专注时长、专注次数、学科/任务分类统计）自动记录与可视化呈现，生成周/月学习报告，帮助用户清晰掌握自身学习节奏与进度。
4. 习惯养成目标：通过任务拆解、目标设定、完成打卡等功能，结合温和的提醒机制，引导用户建立规律的学习习惯，提升学习持续性与成就感。

## 核心功能

**一、番茄计时器：专注状态高效管理**

- 基础适配：默认遵循“25分钟专注+5分钟短休息”“4个循环+15-20分钟长休息”的经典番茄工作法，同时支持自定义专注/休息时长、循环次数，适配不同学习场景（如深度思考、碎片化学习）。
- 专注增强：开启专注模式后自动屏蔽弹窗通知，提供白噪音（如雨声、图书馆环境音）辅助沉浸，支持强制锁定功能（防止中途退出），并记录每次专注的任务标签（如“数学刷题”“英语背诵”）。
- 灵活适配：支持暂停、续时、跳过休息等快捷操作，计时结束通过温和铃声/震动提醒，避免突兀干扰。

**二、日程规划：学习任务系统化拆解**

- 任务创建：支持按日/周/月维度添加学习任务，可设置任务名称、目标时长、优先级（高/中/低）、截止日期及关联学科/分类，支持子任务拆分（如“高数复习”拆分为“知识点梳理+习题练习+错题复盘”）。
- 时间关联：任务可直接绑定番茄计时，点击任务即可启动对应时长的专注模式，完成后自动标记“已完成”，未完成任务支持自动顺延至次日或手动调整。
- 可视化呈现：通过日历视图直观展示每日/每周任务分布，支持拖拽调整任务时间，未完成任务标红提醒，帮助用户快速掌握任务进度与时间分配合理性。

**三、记忆曲线回顾：学习效果精准沉淀**

- 数据追踪：自动记录用户历史专注数据，包括各学科/任务的总专注时长、专注频次、平均专注时长，按日/周/月维度生成趋势图表。
- 记忆适配：基于艾宾浩斯记忆曲线，针对标注为“背诵类”“记忆类”的任务（如单词、知识点），自动生成复习提醒（如学习后1天、3天、7天、14天），用户完成复习后同步记录复习状态。
- 回顾可视化：通过折线图、饼图展示学习时长分布、学科占比、复习完成率，支持导出学习报告，帮助用户复盘学习规律、优化时间分配。

**四、未来功能：AI学习建议（规划中）**

- 个性化分析：基于用户的专注数据、任务完成情况、记忆曲线反馈，AI自动分析学习优势与短板（如“数学专注时长充足但复习频率不足”“碎片化时间未充分利用”）。
- 智能建议：针对分析结果生成定制化建议，包括任务优先级调整、番茄时长优化（如“深度任务建议延长至30分钟”）、复习计划补充，甚至推荐适配学习节奏的资源（如错题本工具、知识点总结资料）。
- 动态适配：随着用户学习习惯变化持续优化建议逻辑，逐步贴合用户个性化学习节奏，助力效率迭代升级。
